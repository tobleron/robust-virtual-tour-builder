## 🔴 CRITICAL ISSUES

### 1. **Path Traversal Vulnerability in Backend** 
**Location:** `backend/src/handlers.rs:594`  
**Severity:** HIGH  
**Risk:** Arbitrary file write/directory traversal

```rust
let sanitized_name: String = fname.replace("..", "").replace("/", "_");
```

**Issue:** The sanitization is insufficient. An attacker could craft filenames like:
- `....//etc/passwd` → becomes `..etc/passwd` (still traverses)
- `..\/..\/etc/passwd` (mixed separators)
- Unicode normalization attacks (e.g., `\u002e\u002e\u002f`)

**Exploit Scenario:**
```javascript
// Malicious project upload
const formData = new FormData();
formData.append('file', blob, '....//etc/shadow');
```

**Recommendation:**
```rust
use std::path::{Path, PathBuf};

fn sanitize_filename(fname: &str) -> Result<String, String> {
    let path = Path::new(fname);
    
    // Reject absolute paths
    if path.is_absolute() {
        return Err("Absolute paths not allowed".to_string());
    }
    
    // Reject paths with parent directory components
    for component in path.components() {
        if matches!(component, std::path::Component::ParentDir) {
            return Err("Parent directory traversal not allowed".to_string());
        }
    }
    
    // Extract only the filename (no directory structure)
    path.file_name()
        .and_then(|s| s.to_str())
        .map(|s| s.to_string())
        .ok_or_else(|| "Invalid filename".to_string())
}
```

### 2. **Unbounded Memory Allocation**
**Location:** `backend/src/handlers.rs:388, 467, 514, 751`  
**Severity:** HIGH  
**Risk:** Denial of Service (OOM)

```rust
let mut data = Vec::with_capacity(32 * 1024 * 1024);
while let Some(mut field) = payload.try_next().await? {
    while let Some(chunk) = field.try_next().await? {
        data.extend_from_slice(&chunk); // ⚠️ No size limit!
    }
}
```

**Issue:** An attacker can upload arbitrarily large files, causing memory exhaustion.

**Exploit Scenario:**
```bash
# Upload 10GB file to crash server
dd if=/dev/zero of=huge.jpg bs=1M count=10240
curl -X POST http://localhost:8080/optimize-image -F "file=@huge.jpg"
```

**Recommendation:**
```rust
const MAX_UPLOAD_SIZE: usize = 100 * 1024 * 1024; // 100MB limit

let mut data = Vec::with_capacity(32 * 1024 * 1024);
let mut total_size = 0;

while let Some(mut field) = payload.try_next().await? {
    while let Some(chunk) = field.try_next().await? {
        total_size += chunk.len();
        if total_size > MAX_UPLOAD_SIZE {
            return Err(AppError::ImageError(
                format!("Upload exceeds maximum size of {}MB", MAX_UPLOAD_SIZE / 1024 / 1024)
            ));
        }
        data.extend_from_slice(&chunk);
    }
}
```

### 3. **Command Injection Risk in FFmpeg Handler**
**Location:** `backend/src/handlers.rs:800`  
**Severity:** HIGH  
**Risk:** Remote Code Execution

```rust
let output = Command::new(&ffmpeg_cmd)
    .arg("-i").arg(&input_str)  // ⚠️ User-controlled path
    .arg("-c:v").arg("libx264")
    // ...
```

**Issue:** While the input path is generated by the server, if an attacker can control the filename through the upload process, they might inject shell metacharacters.

**Current Protection:** Paths are generated via `Uuid::new_v4()`, which is safe.

**Recommendation:** Add explicit validation:
```rust
// Validate that paths don't contain shell metacharacters
fn validate_path(path: &str) -> Result<(), String> {
    if path.contains(&['&', '|', ';', '$', '`', '\n', '\r'][..]) {
        return Err("Invalid characters in path".to_string());
    }
    Ok(())
}

validate_path(&input_str)?;
validate_path(&output_str)?;
```

```javascript
pendingAdvance = setTimeout(() => {
```

### 5. **XSS Vulnerability in Modal HTML Injection**
**Location:** `src/components/LinkModal.js:74-87`  
**Severity:** MEDIUM  
**Risk:** Cross-Site Scripting

```javascript
${state.scenes.map((s, i) => {
    return `<option value="${s.name}" ${isNext} style="background: #1e293b;">${s.name}</option>`;
}).join("")}
```

**Issue:** Scene names are user-controlled (via labels) and directly injected into HTML without sanitization.

**Exploit Scenario:**
```javascript
// User sets scene label to:
'"><script>alert(document.cookie)</script><"'

// Results in:
<option value=""><script>alert(document.cookie)</script><"">...</option>
```

**Recommendation:**
```javascript
function escapeHtml(unsafe) {
    return unsafe
        .replace(/&/g, "&amp;")
        .replace(/</g, "&lt;")
        .replace(/>/g, "&gt;")
        .replace(/"/g, "&quot;")
        .replace(/'/g, "&#039;");
}

${state.scenes.map((s, i) => {
    const safeName = escapeHtml(s.name);
    return `<option value="${safeName}" ${isNext}>${safeName}</option>`;
}).join("")}
```

### 7. **Weak Duplicate Detection**
**Location:** `src/systems/Resizer.js:36-46`  
**Severity:** MEDIUM  
**Risk:** Hash collision, bypassed duplicate detection

```javascript
const FINGERPRINT_SIZE = 2 * 1024 * 1024; // 2MB
const chunk = file.size > FINGERPRINT_SIZE ? file.slice(0, FINGERPRINT_SIZE) : file;
```

**Issue:** Only hashing the first 2MB allows attackers to upload duplicates with different trailing data.

**Exploit Scenario:**
```javascript
// Two files with identical first 2MB but different endings
// Will have same hash but different content
```

**Recommendation:**
```javascript
// Use full file hash for small files, or sample-based hash for large files
export async function getChecksum(file) {
    if (file.size <= 10 * 1024 * 1024) { // 10MB threshold
        // Hash entire file for small files
        const arrayBuffer = await file.arrayBuffer();
        const hashBuffer = await crypto.subtle.digest('SHA-256', arrayBuffer);
        return arrayToHex(hashBuffer) + `_${file.size}`;
    } else {
        // Sample-based hash: beginning + middle + end
        const chunkSize = 1024 * 1024; // 1MB
        const chunks = [
            file.slice(0, chunkSize),
            file.slice(Math.floor(file.size / 2), Math.floor(file.size / 2) + chunkSize),
            file.slice(file.size - chunkSize, file.size)
        ];
        
        const combined = await Promise.all(chunks.map(c => c.arrayBuffer()));
        const concatenated = new Uint8Array(combined.reduce((acc, buf) => acc + buf.byteLength, 0));
        let offset = 0;
        combined.forEach(buf => {
            concatenated.set(new Uint8Array(buf), offset);
            offset += buf.byteLength;
        });
        
        const hashBuffer = await crypto.subtle.digest('SHA-256', concatenated);
        return arrayToHex(hashBuffer) + `_${file.size}`;
    }
}
```

### 9. **Potential Integer Overflow**
**Location:** `backend/src/handlers.rs:275`  
**Severity:** LOW  
**Risk:** Incorrect calculations

```rust
let lum = ((pixel[0] as u32 * 54 + pixel[1] as u32 * 183 + pixel[2] as u32 * 19) >> 8) as u8;
```

**Issue:** While unlikely with 8-bit RGB values, the calculation could theoretically overflow if input validation is bypassed.

**Recommendation:**
```rust
let lum = ((pixel[0] as u32 * 54)

    .saturating_add(pixel[2] as u32 * 19) >> 8) as u8;
```

### 11. **Inconsistent State Management**
**Location:** `src/store.js:294, 307`  
**Issue:** Silent updates without notification

```javascript
updateHotspotTargetView(sceneIndex, hotspotIndex, yaw, pitch, hfov) {
    // ...
    // Do NOT notify to prevent viewer reload. Silent update.
}
```

**Problem:** This creates inconsistency where some state changes trigger re-renders and others don't, making debugging difficult.

**Recommendation:** Implement a flag-based notification system:
```javascript
updateHotspotTargetView(sceneIndex, hotspotIndex, yaw, pitch, hfov, silent = true) {
    const hotspot = this.state.scenes[sceneIndex]?.hotspots[hotspotIndex];
    if (hotspot) {
        hotspot.targetYaw = yaw;
        hotspot.targetPitch = pitch;
        hotspot.targetHfov = hfov;
        if (!silent) this.notify();
    }
}
```
