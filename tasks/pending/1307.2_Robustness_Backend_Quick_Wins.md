# [1307.2] Robustness Hardening: Tier 2 Backend Quick Wins

## Context
Part of the broader Robustness Hardening initiative (Task 1307). These are high-impact fixes for the Rust backend to eliminate potential panics, zombie processes, and race conditions.

## Objectives

### Fix 2.1: Backend Auth expect() Panics
- **File:** `backend/src/auth.rs`
- **Issue:** `env::var("JWT_SECRET").expect("JWT_SECRET must be set")` panics at runtime if env var is missing. Panic kills the worker thread. Any request to a protected endpoint becomes a DoS vector.
- **Fix:** Replace with `env::var("JWT_SECRET").map_err(|_| AppError::InternalError("JWT_SECRET not configured".into()))?`

### Fix 2.2: FFmpeg/Chrome Zombie Process Leak
- **File:** `backend/src/api/media/video_logic.rs`
- **Issue:** No `kill()` on error paths and no timeout on `child.wait()`. If the recording loop exits early or Chrome hangs, FFmpeg processes accumulate as zombies. Browser processes leak on error paths.
- **Fix:**
    1. Add `child.kill()` before early returns
    2. Use `tokio::time::timeout(Duration::from_secs(120), child.wait())` instead of bare `child.wait()`
    3. Add `drop(browser)` explicitly on error paths with `kill()` of Chrome process

### Fix 2.3: Backend Quota Check-Then-Act Race
- **File:** `backend/src/middleware.rs`
- **Issue:** `can_upload()` and `register_upload()` are separate calls. N concurrent requests can all pass the quota check before any registers, bypassing limits by ~N * max_concurrent_threads.
- **Fix:** Combine into atomic `try_register_upload()` that checks AND reserves in one RwLock write. Return the upload_id or an error.

## Verification
- Run `cd backend && cargo test` to ensure no regressions.
- Run `cd backend && cargo clippy` to ensure no new warnings.
