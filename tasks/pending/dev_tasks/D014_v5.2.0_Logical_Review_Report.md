## 1. đź›‘ Loophole: EXIF Local Fallback Hang
**Problem:** `ExifReportGeneratorLogicExtraction.res` still contains a local extraction fallback if metadata is missing. For 40MB+ images, this re-reads the file from disk, causing the "Wrapping Up" hang we tried to avoid.
**Solution:** Implement "Strict Extraction Mode" where files missing backend metadata are simply skipped for reporting/titling rather than re-read locally.
**Task:** `tasks/pending/T1785_fix_exif_fallback_hang.md`

## 2. âšˇ Race Condition: Concurrent Title Discovery
**Problem:** If multiple upload batches finish in rapid succession, `triggerBackgroundTitleDiscovery` will fire multiple times. The `isDiscoveringTitle` global flag will be cleared by the first one to finish, even if others are still working.
**Solution:** Switch `isDiscoveringTitle` to a reference counter or use a serial queue for title discovery tasks.
**Task:** `tasks/pending/T1786_fix_title_discovery_race.md`

## 3. đź“± Edge Case: OffscreenCanvas Browser Support
**Problem:** `ImageOptimizer.res` relies on the `WorkerPool` using `OffscreenCanvas`. If a user is on an older browser (Safari < 16.4, old Chrome), the compression will fail and log an error, but there is no main-thread fallback.
**Solution:** Implement a graceful main-thread `Canvas` fallback in `ImageOptimizer.res` if the `WorkerPool` returns an error or is unsupported.
**Task:** `tasks/pending/T1787_add_image_optimization_fallback.md`

## 4. đź§© Logical Inconsistency: Hotspot Index Stability
**Problem:** `retargetHotspot` state uses `sceneIndex` and `hotspotIndex`. While `LinkModal` is modal-blocked, any background state updates (e.g., from a concurrent collaborative session if implemented later) could invalidate these indices.
**Solution:** Switch `linkInfo` to use `sceneId` and a unique `hotspotId` (uuid) instead of array indices for absolute targeting.
**Task:** `tasks/pending/T1788_refactor_hotspot_targeting_to_ids.md`

## 5. đź“‰ Performance: Worker Memory Spike
**Problem:** `WorkerPool` spawns up to 8 workers. With 12K panoramas, each worker might consume 300MB+ RAM during processing. Total spike could exceed 2GB, potentially crashing low-end devices.

**Task:** `tasks/pending/T1789_optimize_worker_pool_memory.md`
