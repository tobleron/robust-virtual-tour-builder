# 🐛 Bug Report: Tour Preview Transition Block

## 🚨 Symptoms
- **User Impact**: In Tour Preview mode, when the first scene finishes its timeline (or manual navigation is attempted), the tour fails to transition to the next scene.
- **Visuals**: The viewer may hang, show a black screen, or remain on the current scene with the interaction blocked.
- **Recent Context**: A fix was merged (`fix-tour-preview-hang`) to handle timeouts, but logs suggest a deeper crash or resource access issue is triggering this state.

## 🔍 Diagnostic Evidence

### 1. SceneLoader Error (Critical)
The application fails to access the scene image, triggering a critical error.
```log
2026-02-02T11:09:53.771508Z ERROR frontend: SceneLoader LOAD_ERROR 
module=SceneLoader priority=Critical 
json_data=Some(Object {"error": String("The file <a href=\"http://localhost:8080/api/project/.../file/154407_002.webp?token=dev-token\">...</a> could not be accessed.")})
```
*Hypothesis*: The backend might be rejecting the token, the file is missing, or the path is malformed.

### 2. ErrorBoundary Crash
Shortly before or after the load error, an unhandled JS exception occurs, likely crashing the React render cycle or a subsystem.
```log
2026-02-02T11:09:38.978184Z ERROR frontend: ErrorBoundary Cannot set properties of undefined (setting 'e1681cee1698e1ccc1a043f6516c1ad7ebd44102cfe992723e22debda5ae142a_39799516') 
module=ErrorBoundary priority=Critical 
```
*Hypothesis*: This obscure property assignment error suggests an issue with:
- `HotspotLine` or `NavigationRenderer` trying to update a cached element that doesn't exist.
- A hash-based property name generated by ReScript/serialization that is failing.

## 🛠️ Proposed Solution Plan

### Phase 1: Investigation
1. **Reproduce**: Run Tour Preview with the dev console open.
2. **Backend Check**: Verify if `http://localhost:8080/api/project/.../154407_002.webp` truly exists and if the token is valid.
3. **Frontend Trace**: Locate the source of `Cannot set properties of undefined`.
   - Check `NavigationRenderer.res` or `HotspotLine.res` for unsafe object mutations.

### Phase 2: Fix Implementation
1. **Handle 404s Gracefully**: Ensure `SceneLoader` transitions the FSM to `Error` state (handled by recent fix), but *also* ensure the UI shows a visible error instead of crashing.
2. **Fix JS Crash**: Guard the property assignment causing the `ErrorBoundary` trigger.
3. **Verify Transition**: Ensure that even if a scene fails to load, the tour can either skip it or show a "Retry" prompt, rather than blocking indefinitely.

### Phase 3: Validation
- Run the "Tour Preview" journey test.
- Verify `NavigationFSM` correctly enters `Error` state on 404.
